# คู่มือการจัดการผู้ใช้ (User Management Guide)

## 🎯 ภาพรวม

ระบบรองรับการเพิ่มผู้ใช้ 2 วิธี:
1. **เพิ่มผ่าน Admin Panel** (ไม่ต้องมี LINE ID ก่อน)
2. **ลงทะเบียนอัตโนมัติ** (เมื่อผู้ใช้เปิด Mini App ครั้งแรก)

---

## 📝 วิธีที่ 1: เพิ่มผู้ใช้ผ่าน Admin Panel

### ขั้นตอน:

1. **เข้าหน้า Admin → Users**
   - คลิกปุ่ม "+ เพิ่มผู้ใช้"

2. **กรอกข้อมูล:**
   - **ชื่อที่แสดง** (บังคับ): เช่น "คุณแม่สมศรี", "ครูเบียร์"
   - **LINE User ID** (ไม่บังคับ): ปล่อยว่างได้ — จะผูกอัตโนมัติภายหลัง
   - **บทบาท**: ผู้ปกครอง / ครู
   - **สถานะ**: ใช้งาน / ปิดใช้งาน

3. **บันทึก**

4. **ผูกลูก (สำหรับผู้ปกครอง):**
   - คลิกปุ่ม "ผูกลูก" ในแถวของผู้ปกครอง
   - เลือกนักเรียนที่ต้องการผูก
   - บันทึก

5. **แจ้งผู้ปกครอง:**
   - ให้ผู้ปกครองเปิด Mini App ผ่าน LINE
   - ระบบจะผูก LINE ID อัตโนมัติ (ตาม display_name)

---

## 🔄 วิธีที่ 2: ลงทะเบียนอัตโนมัติ

เมื่อผู้ใช้เปิด Mini App ครั้งแรก:

### กรณีที่ 1: มี User ที่รอผูก LINE ID อยู่แล้ว
- ระบบจะค้นหา user ที่:
  - `line_user_id` เป็น `NULL`
  - `display_name` ตรงกับชื่อใน LINE Profile
- ถ้าเจอ → **ผูก LINE ID เข้ากับ user นั้น**
- ผู้ใช้สามารถเข้าระบบได้ทันที พร้อมข้อมูลลูกที่ผูกไว้

### กรณีที่ 2: ไม่มี User ที่รอผูก
- สร้าง user ใหม่อัตโนมัติ:
  - `role`: parent
  - `status`: active
  - `display_name`: จาก LINE Profile
  - `picture_url`: จาก LINE Profile
- **ยังไม่มีลูกผูก** → รอ admin ผูกลูกให้

---

## 🎨 UI Indicators

### สถานะ LINE ID:
- ✅ **"✓ Uxxxxxxx..."** (สีเขียว) → ผูก LINE แล้ว
- ⏳ **"⏳ รอผูก LINE"** (สีส้ม) → ยังไม่ผูก LINE

### สถานะลูก:
- ✅ **ชื่อลูก** (พื้นเขียว) → ผูกลูกแล้ว
- ⚠️ **"⚠️ ยังไม่ผูก"** (พื้นเหลือง) → ยังไม่ผูกลูก

---

## 🔧 Workflow แนะนำ

### สำหรับผู้ปกครองใหม่:

```
1. Admin เพิ่มผู้ใช้ (ใส่ชื่อ "คุณแม่สมศรี")
   ↓
2. Admin ผูกลูก (เลือกนักเรียน "น้องมะลิ")
   ↓
3. แจ้งผู้ปกครองให้เปิด Mini App
   ↓
4. ผู้ปกครองเปิด Mini App (ชื่อ LINE = "คุณแม่สมศรี")
   ↓
5. ระบบผูก LINE ID อัตโนมัติ
   ↓
6. ผู้ปกครองเห็นรายงานของ "น้องมะลิ" ทันที ✅
```

### สำหรับครูใหม่:

```
1. Admin เพิ่มผู้ใช้ (ใส่ชื่อ "ครูเบียร์", role = teacher)
   ↓
2. แจ้งครูให้เปิด Mini App
   ↓
3. ครูเปิด Mini App (ชื่อ LINE = "ครูเบียร์")
   ↓
4. ระบบผูก LINE ID อัตโนมัติ
   ↓
5. ครูเข้าหน้า Admin ได้ทันที ✅
```

---

## ⚠️ ข้อควรระวัง

1. **ชื่อต้องตรงกัน:**
   - ชื่อที่กรอกใน Admin Panel ต้องตรงกับชื่อใน LINE Profile
   - ถ้าไม่ตรง → ระบบจะสร้าง user ใหม่แทน

2. **ผูกลูกก่อนแจ้งผู้ปกครอง:**
   - ควรผูกลูกให้เรียบร้อยก่อนแจ้งผู้ปกครองเปิด Mini App
   - ถ้าผูกทีหลัง → ผู้ปกครองต้อง refresh หน้าเว็บ

3. **LINE ID ไม่ซ้ำ:**
   - 1 LINE ID = 1 User เท่านั้น
   - ถ้าผูกแล้ว จะไม่สามารถผูกซ้ำได้

---

## 🔍 การแก้ปัญหา

### ปัญหา: ผู้ปกครองเปิด Mini App แล้วไม่เห็นลูก

**สาเหตุ:**
- ชื่อใน LINE ไม่ตรงกับชื่อใน Admin Panel
- ระบบสร้าง user ใหม่แทนที่จะผูกกับ user เดิม

**วิธีแก้:**
1. ตรวจสอบว่ามี user ซ้ำหรือไม่ (ค้นหาชื่อใน Admin → Users)
2. ลบ user ที่สร้างใหม่ (ที่ยังไม่มีลูกผูก)
3. แก้ไขชื่อใน user เดิมให้ตรงกับ LINE Profile
4. ให้ผู้ปกครองเปิด Mini App อีกครั้ง

### ปัญหา: ต้องการผูก LINE ID ด้วยตนเอง

**วิธีทำ:**
1. ให้ผู้ใช้เปิด Mini App ครั้งแรก
2. ดู LINE User ID จาก Console / Log
3. กลับมาที่ Admin → Users → แก้ไข user
4. วาง LINE User ID ลงในช่อง "LINE User ID"
5. บันทึก

---

## 📊 Database Schema

```sql
CREATE TABLE app_user (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  line_user_id TEXT UNIQUE,  -- NULL = ยังไม่ผูก LINE
  display_name TEXT,
  picture_url TEXT,
  role TEXT NOT NULL,        -- 'parent' | 'teacher'
  status TEXT NOT NULL,      -- 'active' | 'inactive'
  created_at TIMESTAMP DEFAULT NOW()
);

-- NULL line_user_id ไม่ถือว่าซ้ำ (PostgreSQL UNIQUE constraint)
-- สามารถมีหลาย user ที่ line_user_id = NULL ได้
```
